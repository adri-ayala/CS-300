# CS-300

# What was the problem you were solving in the projects for this course?
The goal of these projects was to build a course advising system that could store course information, sort it in alphanumeric order, and allow users to look up individual courses and their prerequisites. The main challenge was choosing the right data structure to make sorting and searching efficient.

# How did you approach the problem? Consider why data structures are important to understand.
I focused on selecting a data structure that naturally supports sorting and searching. I chose a Binary Search Tree because it allows in-order traversal to print courses in sorted order and makes lookups efficient. This project reinforced how important data structures are — the right structure makes the implementation much simpler and more efficient.

# How did you overcome any roadblocks you encountered while going through the activities or project?
The most significant obstacles to my success were creating a clean design for the data I need to work with and implementing that design simply while keeping as much complexity out as possible. To break this down, I created smaller pieces of work to complete (loading data, inserting data into the tree, searching data, printing data). I tested each of those pieces individually before combining them to produce the full solution.

# How has your work on this project expanded your approach to designing software and developing programs?
I thought more about the structure of the code before I jumped into implementing it. This has also changed my overall process for creating software because I will now evaluate runtime, memory use, and overall software design before I begin implementing a solution. The planning process has also contributed to the smoothness of creating a final product after the solution has been fully implemented.

# How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?
This has helped to push me towards writing cleaner code with more attention given to organization ultimately improving the way the code will be maintained in the future. I focused on making my code readable, on meaningful separation of functions, and maintaining simple logic. I also became aware of how the structures and patterns I was using to implement my code would contribute to how I would maintain the code in the future.
